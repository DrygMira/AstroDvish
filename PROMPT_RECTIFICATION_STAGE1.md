Фазовый flow:
1) phase="element_detection": первые 5-6 вопросов на стихию.
2) phase="modality_detection": 3-4 вопроса на крест внутри лидирующей стихии.
3) phase final_result: итог по стихии/кресту/знаку.

Логика финализации:
- если mode == "finalize_now": вернуть final_result;
- если step_count >= max_steps: вернуть final_result;
- иначе если данных недостаточно: ask_question;
- если кандидаты близки, явно укажи это и поставь needs_more_questions=true.

Требования к final_result:
- primary_candidate: знак + интервалы времени;
- secondary_candidates: ближайшие альтернативы + интервалы;
- element_scores, modality_scores, sign_scores;
- leading_element, leading_modality;
- candidate_group при близких кандидатах;
- method_limitations (короткий список ограничений);
- warnings (если есть пограничность/близость).
